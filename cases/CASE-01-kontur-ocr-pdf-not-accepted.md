# CASE-01 — Kontur OCR: PDF не принимается в модалке распознавания паспорта (upload silent fail)

**Project:** CRM  
**Модуль:** Контур (CV) → Распознавание паспорта (модалка)  
**Дата кейса:** 30.10.2025  
**Окружение:** test · `v.689ce285` · admin · Windows 11 / Chrome / 1920×1080  

## Контекст
В инструменте **Контур (CV) → Распознавание паспорта** пользователь загружает файл первой страницы паспорта для распознавания.  
По бизнес-ожиданию поведение должно быть **унифицировано** с разделом **«Документы клиента»**, где PDF принимается.

## Что тестировала
- Инструмент: **Контур (CV) → Распознавание паспорта (модалка «Проверить физ. лицо»)**.
- Сценарий загрузки файла первой страницы паспорта в формате **PDF**.
- Поведение UI и запуск распознавания (в т.ч. реакция без явной ошибки).

## Что нашла / что покрыла
- **PDF (одностраничный) не принимается**: загрузка игнорируется, распознавание не запускается.
- Зафиксировала **стабильную воспроизводимость** и корректное окружение.
- Сформулировала развилку требований:
  1. поддержать PDF “как в «Документах клиента»”, либо
  2. явно запретить PDF с понятным текстом ошибки (согласовать ТЗ).
- Добавила **acceptance criteria** для фикса и ретеста (поддержка/ясная ошибка, стабильность модалки, отсутствие JS-ошибок, унификация поведения).

## Чем полезно команде
- Убирает “тихий” отказ без обратной связи и снижает количество обращений/ошибок пользователей.
- Выравнивает UX и правила загрузки между модулями (единое поведение продукта).
- Даёт разработчику проверяемые критерии для фикса, а QA — понятный план ретеста.

Сформирован дефект с чёткой развилкой “поддерживаем vs запрещаем”, привязанный к бизнес-ожиданию и унификации поведения.  
Есть проверяемые acceptance для фикса и дальнейшего ретеста.
