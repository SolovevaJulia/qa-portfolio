# CASE-02 — Cold Calls (CRM): SMOKE-сессия + дефекты + план ретеста

**Project:** CRM  
**Модуль:** Инструменты → Холодные звонки  
**Дата кейса:** 14.01.2026  
**Окружение:** test · `v.539de326` · admin · Windows 11 / Chrome / 1920×1080  

## Контекст
Модуль **CRM → Инструменты → Холодные звонки** проверялся с целью быстро оценить жизнеспособность основного флоу:
**таблица → создание → карточка → редактирование/удаление**, а также выявить блокирующие проблемы до активного использования.

## Что тестировала
- Модуль **CRM → Инструменты → Холодные звонки** (SMOKE).
- Ключевые потоки: **таблица → создание → карточка → редактирование/удаление**.
- Поиск/фильтры/пагинация, отображение длинных значений, валидации (URL/цена/тексты).
- Блок **«Полезные советы»** (переносы/эмодзи + проверка XSS: `<script>alert(1)</script>` как текст).
- Network/консоль на ошибки 422/500 и “тихие” отказы.

## Что нашла / что покрыла
- **404 в сценариях валидации/бизнес-логики при создании** (невалидный URL / повторный клиент) — вместо понятной ошибки для пользователя.
- **Фильтр по «Создан» не работает.**
- **Ошибки валидации непонятные/не локализованы.**
- **При 2+ «полезных советах» в таблице не отображается ни один** (данные есть, но UI вводит в заблуждение).
- Дополнительно: опечатки/тексты (“Нет найдено” → “Не найдено”), корректность заголовков колонок.
- Зафиксировала итог сессии в формате **«Итоги проверки»**: покрытие, найдено, риски и план ретеста + ретест-лог.

## Чем полезно команде
- Раннее выявление блокеров SMOKE до выхода в активное использование (экономит время команды и снижает инциденты).
- Даёт понятный список проблем, которые напрямую бьют по UX и рабочему процессу пользователей.
- План ретеста ускоряет проверку фиксов и снижает риск регресса в соседних сценариях.
