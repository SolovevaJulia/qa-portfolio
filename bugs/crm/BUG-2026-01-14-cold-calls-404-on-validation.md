# BUG-2026-01-14 — Холодные звонки: при создании возвращается 404 в валидационных/бизнес-сценариях (невалидная ссылка / повторный клиент)

**Project:** CRM  
**Компонент/Модуль:** Инструменты → Холодные звонки  
**Экран/URL/Маршрут:** https://crm-test.ru (форма: «Добавить»)  
**Reporter:** Юлия  
**Окружение:** **test**  
**Сборка/Версия:** **v.539de326**  
**Роль:** **admin**  
**ОС/Браузер/Разрешение:** **Windows 11 / Chrome (последняя) / 1920×1080**  
**Повторяемость:** всегда  
**Частота:** 5/5

## Шаги для воспроизведения
Сценарий A — невалидная ссылка:
1. Открыть CRM → **Инструменты → Холодные звонки**.
2. Нажать **«Добавить»**.
3. Выбрать клиента.
4. В поле **«Ссылка на объект»** ввести невалидное значение (например, `asdf`).
5. Нажать **«Сохранить»**.

Сценарий B — повторный клиент:
1. Открыть CRM → **Инструменты → Холодные звонки** → **«Добавить»**.
2. Выбрать клиента, который уже использовался в другом холодном звонке.
3. Нажать **«Сохранить»**.

## Ожидаемый результат
- Сценарий A: валидация на UI у поля «Ссылка на объект» + корректный статус (например, 400/422), без 404.
- Сценарий B: понятная ошибка бизнес-правила (например, 409/422) с текстом «клиент уже используется…» (если правило действительно запрещает), без 404.

## Фактический результат
- В обоих сценариях при сохранении отображается ошибка **404**.

## Доп. детали
- 404 трактуется как «не найдено», но по сути это сценарии валидации/бизнес-правил → пользователю непонятно, что исправлять.
- Желательно вернуть корректные статусы и человекочитаемые сообщения.

## Acceptance для фикса
- Валидационные ошибки возвращают корректный статус (400/422) и текст, привязанный к полю.
- Бизнес-конфликт (повторный клиент) — 409/422 + понятное сообщение (если ограничение реально).
- Нет 404 на корректных путях формы создания.
